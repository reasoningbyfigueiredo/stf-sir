# STF-SIR Benchmark Report Format — v1

**Format identifier:** `stf-sir-benchmark-v1`

## Overview

A `BenchmarkReport` captures the results of running the STF-SIR compiler on a document corpus
and computing ρ_v2 retention metrics. Reports are serializable to JSON/YAML and are suitable
for CI artifact storage and audit trails.

---

## Fields

| Field | Type | Description |
|---|---|---|
| `format` | `String` | Always `"stf-sir-benchmark-v1"`. Version sentinel. |
| `corpus_id` | `String` | Identifier for the benchmark corpus (e.g., `"stf-sir-v2-corpus"`). |
| `compiler_version` | `String` | Version of the STF-SIR compiler used (from `CARGO_PKG_VERSION`). |
| `timestamp` | `String` | ISO 8601 timestamp of when the benchmark was run. |
| `aggregate` | `SerializableAggregateMetrics` | Aggregate statistics over the corpus. |
| `retention_v2` | `RetentionV2Score` | ρ_v2 scores for the corpus. |

### SerializableAggregateMetrics fields

| Field | Type | Description |
|---|---|---|
| `total_documents` | `usize` | Total number of documents in the corpus. |
| `successful_compilations` | `usize` | Number of documents that compiled without error. |
| `mean_token_count` | `f64` | Mean token count across successfully compiled documents. |
| `mean_retention_v2` | `f64` | Mean composite ρ_v2 score across all documents. |

### RetentionV2Score fields

See [retention-v2.md](retention-v2.md) for full component definitions.

| Field | Type | Description |
|---|---|---|
| `rho_l` | `f64` | Lexical retention [0.0, 1.0] |
| `rho_s` | `f64` | Syntactic retention [0.0, 1.0] |
| `rho_sigma_gloss` | `f64` | Semantic gloss retention [0.0, 1.0] |
| `rho_sigma_concepts` | `f64` | Semantic concepts retention [0.0, 1.0] |
| `rho_phi` | `f64` | Logical retention [0.0, 1.0] |
| `rho_corpus` | `f64` | Corpus-level aggregate [0.0, 1.0] |

---

## Invariants

- **Reproducibility:** identical corpus + identical compiler binary → byte-identical ρ_v2 scores.
- **Bounds:** all ρ components are in [0.0, 1.0].
- **No silent failures:** a compile error causes the entry to be recorded as `compile_success: false` — it is never silently skipped.

## Drift Detection

A `DriftReport` can be generated by comparing two `BenchmarkReport` instances with
`DriftDetector::detect`. Drift is flagged when any ρ component regresses by more
than the configured threshold (default: 2%).

```rust
use stf_sir::benchmark::DriftDetector;

let detector = DriftDetector::new(0.02);
let drift = detector.detect(&baseline_report, &current_report);
if drift.detected {
    eprintln!("Retention regression detected!");
}
```

## Usage

```rust
use stf_sir::benchmark::{BenchmarkHarness, CorpusEntry};

let harness = BenchmarkHarness::new("my-corpus");
let corpus = vec![
    CorpusEntry {
        document_id: "doc-001".to_string(),
        source: "# Title\n\nContent.\n".to_string(),
        expected_token_count: None,
    },
];
let corpus_report = harness.run(&corpus);
println!("Successful: {}/{}", corpus_report.aggregate.successful_compilations, corpus_report.aggregate.total_documents);
```
